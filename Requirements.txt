Role: You are an Expert EdTech Full-Stack Developer and UI/UX Designer.
Objective: Design the architecture, data models, and core logic for a gamified math learning Progressive Web App (PWA) called PebbleSum. The UI and UX should be specifically calibrated to be highly engaging for ages ranging from 2-year-old to 15-year-old. The app's core philosophy is built on daily repetition, strict time-based mastery, and progressive difficulty, utilizing a Kumon-style mathematical progression.
1. Architecture & Tech Stack
Platform: A Progressive Web App (PWA) accessible via a direct browser URL and downloadable to the device. The codebase (e.g., Next.js, React, or Vue) must be cleanly decoupled from web-specific APIs so it can easily be wrapped with Capacitor or React Native for future native Android and iOS deployments.
Deployment: Netlify.
Database & Storage: Utilize Netlify Database (Serverless Postgres) for relational data (users, progress, streaks, completion times) and Netlify Blobs for unstructured assets (tutorial images). Configure the database schema to ensure PII is properly hashed and utilize Row Level Security (RLS) to maximize Netlify's built-in at-rest encryption.
2. Authentication & Security
Login Methods: Passwordless authentication using Email or Mobile Number.
Verification: Implement a strict OTP (One-Time Password) verification flow via SMS or Email magic link.
Bot Prevention: Integrate an invisible anti-bot solution (like Cloudflare Turnstile or reCAPTCHA v3) on all authentication and signup routes.
Compliance: The signup/login flow must include a mandatory "Accept Terms and Conditions" checkbox before the verification code is triggered.
3. The Progression Framework (The Curriculum)
The app must follow a strict, linear progression path divided into specific stages. A user cannot access a higher stage without mastering the current one.
Stage 6A - 4A (Motor Skills & Counting): Straight lines, curved lines, basic number counting (up to 100), and number tracing.
Stage 3A - 2A (Basic Addition): Addition preparation and basic addition (from +1 up to +10).
Stage A - B (Advanced Addition & Subtraction): 2-digit addition, subtraction preparation, and 2-digit subtraction.
Stage C - D (Multiplication & Division): Basic multiplication tables, multi-digit multiplication, division with remainders, and multi-digit division.
Stage E - F (Fractions, Decimals, Order of Operations): Adding, subtracting, multiplying, and dividing fractions, decimals, and basic order of operations.
4. Core Features & App Mechanics
The Daily Loop (Duolingo-style): The app generates a daily set of questions based on the user's current stage. The problems must be algorithmically generated (similar but varying) so the method is learned, not memorized. Gamify this with daily streaks and experience points (XP).
Speed & Accuracy Gating (Standard Completion Time): Each stage has a "Standard Completion Time" (SCT). The app must track the time taken to complete the daily set. The user only levels up to the next micro-stage if they achieve 100% accuracy within the SCT threshold for three consecutive days.
Stage Landing Pages (Tutorial Gates): Before unlocking a new concept, the user hits a locked "Landing Page." This page must render a highly visual, pictorial representation of the new concept before the timer begins.

Please provide the following:
System Architecture: A brief outline of the PWA setup and how the frontend components will be structured for future native porting.
Database Schema & Auth Flow: Design the Postgres models to track User Profiles, OTP Verification, Terms Acceptance, Daily Streaks, Completion Times (SCT), and Category Unlocks.
Algorithm Design: Write a sample function (in TypeScript) that algorithmically generates varying but similar math problems for "Stage 2A: Addition (+2 and +3)".
UI/UX Component Tree: Provide a component structure for the "Auth/Login Screen" (incorporating bot prevention and T&Cs) and the "Daily Practice Screen."
User might interact from multiple device types and might even be able to use a pen to enter the answer or data.
